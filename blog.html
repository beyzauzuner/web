GitHub ve Git Rehberi

10 Mart 2026

Günümüz yazılım dünyasında kod yazmak sadece satırlar oluşturmak değil, bu satırları yönetebilmek ve paylaşabilmektir[cite: 2]. İşte bu noktada karşımıza iki dev kavram çıkıyor: Git ve GitHub[cite: 3].

Neden GitHub Kullanmalıyız?

  • Versiyon Kontrolü: Projenizin eski hallerine saniyeler içinde dönebilirsiniz[cite: 9].
  • İş Birliği: Aynı anda birçok kişi tek bir proje üzerinde çalışabilir[cite: 10].
  • Dijital Portfolyo: GitHub profiliniz, yazılımcılar için en etkili özgeçmiş (CV) yerine geçer[cite: 12].

Temel Kavramlar

Repository (Depo): Projenizin tüm geçmişinin tutulduğu dijital kumbaradır[cite: 16].

Commit: Değişiklikleri mühürleyip sisteme kalıcı olarak kaydetmektir[cite: 19].

En Çok Kullanılan Git Komutları

git init

Bulunduğunuz klasörü bir Git deposu haline getirir[cite: 23].

git add .

Değişiklikleri hazırlık alanına (Staging Area) taşır[cite: 39].

git commit -m "mesaj"

Dosyaları kalıcı bir versiyon haline getirir[cite: 43].

git push

Yerel kayıtları GitHub'daki uzak depoya gönderir[cite: 29].
